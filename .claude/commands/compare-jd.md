첨부한 JD와 현재 포트폴리오를 비교해줘.

분석 형식
1. JD 핵심 요구사항 요약
2. 현재 잘 맞는 점
3. 부족한 점
4. 포트폴리오에 추가로 보여줘야 할 점
5. 수정 우선순위 3개

저장 규칙
- 결과는 반드시 $2 파일에 저장한다
- $2는 docs/analysis/jd-gap/ 아래 경로여야 한다
- 저장 파일 상단에는 아래 메타 정보를 포함한다
  - 분석일
  - 분석 대상 JD
  - 분석 종류: jd-gap

주의사항
- 실제 소스 파일은 수정하지 않는다
- 근거 없는 경험, 성과, 수치는 만들지 않는다
- 없는 역량과 이미 했지만 포트폴리오에 드러나지 않은 역량을 구분한다
- 가능하면 어떤 파일의 어떤 부분을 보강해야 하는지도 함께 짚는다
