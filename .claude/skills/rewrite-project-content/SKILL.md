# 프로젝트 설명 재작성 Skill

이 Skill은 project-modal.js 또는 관련 파일에 있는 프로젝트 설명을 재작성할 때 사용한다.

## 목적

길거나 모호한 프로젝트 설명을 채용 친화적인 시니어 iOS 포트폴리오 문구로 바꾼다.

## 재작성 구조

각 프로젝트는 가능하면 다음 구조로 정리한다.

1. Overview
2. Problem
3. My Role
4. Key Contributions
5. Technical Decisions
6. Result / Impact
7. Interview Points

## 작성 규칙

- 주 언어는 한국어로 작성한다.
- 간결하게 쓴다.
- 과장하지 않는다.
- 검증되지 않은 수치를 만들지 않는다.
- 소유 범위를 부풀리지 않는다.
- 회사 배경 설명을 길게 쓰지 않는다.
- 시니어 iOS 개발자 가치가 드러나게 쓴다.
- 아키텍처, 운영, 협업, 기술 판단, 안정성 개선, 플랫폼화 경험을 우선적으로 드러낸다.
- AI 도구를 사용한 경우, 분석/테스트/리팩토링/문서화 보조로 표현한다.

## 피해야 할 표현

- 다양한 업무를 수행했습니다
- 많은 부분에 기여했습니다
- 개발에 참여했습니다
- 여러 기능을 만들었습니다

위 표현 대신 다음처럼 쓴다.

- 어떤 문제를 해결했는지
- 어떤 역할을 맡았는지
- 어떤 기술적 판단을 했는지
- 어떤 구조를 설계했는지
- 어떤 결과를 만들었는지

## 출력 방식

파일 수정 전:

1. 현재 문구의 문제점
2. 재작성 방향
3. 재작성 문구
4. 왜 더 나은지

파일 수정 후:

1. 변경 파일
2. 변경 내용
3. 강화된 포인트
4. 남은 리스크
