# 최종 포트폴리오 리뷰 Skill

이 Skill은 포트폴리오 수정 완료 후 커밋 또는 배포 전에 사용한다.

## 리뷰 목적

리뉴얼된 포트폴리오가 Senior iOS Application Developer 채용 목적에 맞게 완성되었는지 확인한다.

## 채용 전략 체크

- Hero에서 지원자가 누구인지 바로 보이는가?
- 첫 화면에서 Senior iOS 가치가 5초 안에 전달되는가?
- 경력 흐름이 쉽게 읽히는가?
- 대표 프로젝트가 적절히 우선순위화되어 있는가?
- 오래된 프로젝트가 현재 강점을 흐리지 않는가?
- AI 활용 경험이 구체적이고 믿을 만한가?
- 사이드 프로젝트가 실무 역량을 보완하는가?
- CTA가 명확한가?

## 콘텐츠 체크

- 모호한 표현이 없는가?
- 검증되지 않은 수치가 없는가?
- 프로젝트 설명이 너무 길지 않은가?
- 회사명, 직무명, 기간이 일관적인가?
- 같은 주장이 반복되지 않는가?
- 한국어 문체가 전문적이고 간결한가?
- AI 사용이 과장되어 보이지 않는가?

## UI 체크

- 모바일 화면에서 읽기 쉬운가?
- 카드 정렬이 일관적인가?
- 섹션 간 간격이 자연스러운가?
- 모달 내용이 읽기 쉬운가?
- CTA 버튼이 명확한가?
- 다크 테마에서 대비가 충분한가?

## 기술 체크

- 내부 링크가 깨지지 않았는가?
- 이미지 경로가 유효한가?
- JS selector 오류 가능성이 없는가?
- 중복 ID가 없는가?
- class 이름이 일관적인가?
- 의도치 않게 삭제된 콘텐츠가 없는가?

## 출력 방식

1. 최종 판단: 통과 / 수정 필요
2. 치명적인 문제
3. 경미한 문제
4. 추천 수정안
5. 있으면 좋은 개선 사항
6. 배포 가능 여부
