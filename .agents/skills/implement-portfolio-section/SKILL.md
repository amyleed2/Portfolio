# 포트폴리오 섹션 구현 Skill

이 Skill은 포트폴리오에 새 섹션을 추가하거나 기존 섹션을 개선할 때 사용한다.

## 목적

채용 관점에서 필요한 섹션을 정적 웹사이트 구조에 맞게 안정적으로 구현한다.

## 실행 단계

1. index.html과 styles.css를 읽는다.
2. 기존 섹션 구조와 class naming 패턴을 확인한다.
3. 기존 디자인 토큰과 CSS 구조를 가능한 한 재사용한다.
4. 의미 있는 HTML 구조로 작성한다.
5. 모바일 반응형 스타일을 함께 고려한다.
6. 불필요한 JavaScript를 추가하지 않는다.
7. JavaScript가 필요한 경우 이유를 설명한다.
8. anchor link, section id, CTA 버튼 연결을 확인한다.

## 구현 원칙

- 가독성을 우선한다.
- 과한 애니메이션을 피한다.
- 기존 모달 구조를 깨뜨리지 않는다.
- 중복 ID를 만들지 않는다.
- 불필요한 class를 과도하게 만들지 않는다.
- 모바일에서도 섹션 간 간격과 카드 폭이 자연스럽게 보이도록 한다.

## 필수 확인 항목

- 데스크톱 가독성
- 모바일 가독성
- 중복 ID 없음
- 깨진 nav link 없음
- 깨진 modal trigger 없음
- 불필요한 JS 없음
- 기존 프로젝트/경력 모달 동작 유지

## 출력 방식

1. 변경 계획
2. 변경 파일
3. 구현 요약
4. 수동 브라우저 QA 체크리스트
5. 남은 개선 사항
