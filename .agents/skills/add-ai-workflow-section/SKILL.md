# AI Workflow 섹션 추가 Skill

이 Skill은 포트폴리오에 AI 도구 활용 경험을 추가하거나 개선할 때 사용한다.

## 목적

AI 도구 사용 경험을 단순 도구 나열이 아니라, 시니어 iOS 개발자의 실무 생산성 개선 방식으로 보여준다.

## 사용 도구

지원자가 사용한 AI 도구는 다음과 같다.

- Cursor
- Claude Code
- Codex
- Copilot
- ChatGPT
- Perplexity
- Figma Make
- Pencil

## 포지셔닝

AI는 다음 방식으로 표현한다.

- 코드베이스 탐색 보조
- 레거시 코드 흐름 분석 보조
- SwiftUI / MVVM 전환 구조 초안 작성
- 리팩토링 방향 검토
- 테스트 케이스 초안 작성
- 문서화 보조
- 사이드 프로젝트 기획 보조
- UI 콘셉트 탐색 보조

AI를 다음 방식으로 표현하면 안 된다.

- AI가 개발을 대신했다
- AI로 모든 작업을 자동화했다
- AI가 메인 역량이다
- 개발자의 판단 없이 AI 결과를 그대로 사용했다

핵심 메시지는 다음과 같다.

AI는 생산성을 높이는 도구이고, 최종 설계 판단과 품질 검증은 개발자가 수행한다.

## 추천 섹션 제목

기본 제목:

AI-Augmented Development Workflow

한국어 보조 문구:

AI를 단순 코드 생성 도구가 아니라, 설계 검토·리팩토링·테스트·문서화의 생산성을 높이는 보조 도구로 활용합니다.

## 추천 카드 구성

1. Legacy Code Analysis
   - 레거시 UIKit 코드 흐름을 빠르게 파악하고 SwiftUI 전환 범위를 정리한다.

2. Architecture Drafting
   - MVVM, Clean Architecture, DI 구조 초안을 작성하고 개발자가 최종 판단한다.

3. Test Case Drafting
   - 주요 유스케이스와 예외 케이스를 기준으로 테스트 초안을 작성한다.

4. Refactoring Review
   - 변경 범위, 영향도, 리스크를 검토하고 단계적 리팩토링 계획을 세운다.

5. Documentation
   - 설계 의도, 변경 이유, 코드 리뷰 포인트를 문서화한다.

6. Product / UI Ideation
   - 사이드 프로젝트의 화면 흐름, 와이어프레임, 기능 우선순위를 정리한다.

## 출력 방식

1. 제안 문구
2. HTML 구조
3. CSS class 제안
4. 수정 대상 파일
5. 구현 내용
6. 수동 확인 체크리스트
