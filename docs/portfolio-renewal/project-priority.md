# 프로젝트 우선순위 정리

## 목적

포트폴리오에 모든 프로젝트를 동일한 비중으로 나열하지 않기 위해 프로젝트 우선순위를 정리한다.

10년 이상 경력의 iOS 개발자 포트폴리오에서는  
“많이 했다”보다 “무엇을 대표 성과로 보여줄 것인가”가 중요하다.

---

## Featured Projects

### 1. ONEstore Design System

대표 프로젝트로 가장 강하게 보여준다.

## 보여줄 역량

- 디자인 시스템 구축
- 공통 UI 컴포넌트 설계
- 디자인 토큰 관리
- Figma Code Connect 활용
- iOS / Android 협업 구조
- 플랫폼화
- 개발 리딩

## 설명 방향

이 프로젝트는 단순 UI 작업이 아니라  
디자인과 개발 사이의 반복 비용을 줄이고,  
공통 컴포넌트 기반의 개발 체계를 만드는 작업으로 설명한다.

## Early Career로 요약할 프로젝트

### Digital Daum Android / SI 프로젝트

이 프로젝트들은 완전히 삭제하지 않는다.

하지만 현재 목표가 Senior iOS Developer 채용이므로  
대표 프로젝트와 같은 비중으로 보여주지는 않는다.

## 설명 방향

다음처럼 요약한다.

초기 경력에서는 Android 기반 SI/SM 프로젝트를 수행하며  
커머스, 브랜드, O2O, 위치 기반 서비스, 외부 SDK 연동 경험을 쌓았습니다.  
이 시기의 경험은 이후 iOS 개발자로 전환한 뒤에도  
모바일 앱 구조와 운영 흐름을 이해하는 기반이 되었습니다.

## 제외하거나 축소할 기준

다음에 해당하는 프로젝트는 메인에서 제외하거나 축소한다.

- 오래되었고 현재 iOS 역량과 직접 관련이 약한 프로젝트
- 설명은 길지만 기술적 의사결정이 잘 드러나지 않는 프로젝트
- 단순 참여 수준으로 보일 가능성이 큰 프로젝트
- 회사/서비스 소개가 대부분인 프로젝트
- 현재 목표 포지션과 연결이 약한 프로젝트

## 프로젝트 설명 공통 포맷

각 대표 프로젝트는 다음 포맷으로 작성한다.

1. Overview
2. Problem
3. My Role
4. Key Contributions
5. Technical Decisions
6. Result / Impact
7. Interview Points

이 포맷을 지키면 채용 담당자는 빠르게 이해하고,  
현업 면접관은 기술 질문 포인트를 잡기 쉬워진다.
