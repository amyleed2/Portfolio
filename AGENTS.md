## AI 활용 표현 규칙

AI 도구 사용 경험은 반드시 강조한다.  
하지만 AI가 모든 개발을 대신했다는 느낌을 주면 안 된다.

AI는 다음처럼 표현한다.

- 레거시 코드 분석 보조
- 구조 설계 초안 작성 보조
- 테스트 케이스 작성 보조
- 리팩토링 방향 검토
- 문서화 보조
- 반복 작업 생산성 개선
- 사이드 프로젝트 기획 및 화면 시안 작성 보조

AI를 다음처럼 표현하면 안 된다.

- AI가 개발을 대신했다
- AI로 대부분 자동화했다
- AI만으로 프로젝트를 만들었다
- 개발 판단 없이 AI 결과를 그대로 사용했다

반드시 다음 관점을 유지한다.

AI는 생산성을 높이는 도구이며, 최종 설계 판단과 코드 품질 검증은 개발자가 수행한다.

## 코드베이스 작업 규칙

이 프로젝트는 정적 포트폴리오 웹사이트다.

주요 수정 대상 파일은 다음과 같다.

- index.html
- styles.css
- script.js
- career-modal.js
- project-modal.js

파일을 수정하기 전에 반드시 다음을 수행한다.

1. 현재 파일 구조를 읽는다.
2. 수정 대상 파일을 식별한다.
3. 변경 계획을 먼저 설명한다.
4. 필요한 범위만 수정한다.
5. 기존 동작을 불필요하게 깨뜨리지 않는다.

금지 사항:

- 전체 사이트를 무작정 새로 작성하지 않는다.
- 기존 내용을 이유 없이 삭제하지 않는다.
- 검증되지 않은 수치를 만들지 않는다.
- 과장된 성과를 쓰지 않는다.
- 불필요한 JavaScript를 추가하지 않는다.
- 불필요한 프레임워크를 도입하지 않는다.
